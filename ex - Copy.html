HTML Elements

The content enclosed between a tags called HTML elements

Block Level Elements:

A block-level element always starts in a new line and it occupies the whole horizontal space of its parent element & the height is equal to the content's height.

Inline Elements:

An inline element is the opposite of the block-level element. It does not start in a new line and takes up only the necessary instead of breaking the flow of the content.

Difference Between Inline and Block Elements

Inline Elements Block Elements
Inline elements occupy only sufficient width required. Block Elements occupy the full width irrespective of their sufficiency.
Inline elements don’t start in a new line. Block elements always start in a line.
Inline elements allow other inline elements to sit behind. Block elements doesn’t allow other elements to sit behind
Inline elements have top and bottom margin Block elements don’t have top and bottom margin.

Semantic Tags (Elements)

A semantic element clearly describes its meaning to both the browser and the developer.

Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.

Most of semantic elements are block level elements.

Examples of semantic elements: <form>, <table>, and <article> - Clearly defines its content

Semantic elements represents their meaning to both the browser and developer.

Some Semantic Elements Are:

H - HEADER
The <header> element represents a container for introductory content or a set of navigational links.
D - DETAILS
The <details> tag specifies additional details that the user can open and close on demand. The <details> tag is often used to create an interactive widget that the user can open and close. By default, the widget is closed. When open, it expands, and displays the content within. Any sort of content can be put inside the <details> tag.
By summary tag we can change the name of details tag also
A - ASIDE
The <aside> tag defines some content aside from the content it is placed in.
M - MAIN
The <main> tag specifies the main content of a document. The content inside the <main> element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.
N - NAV
The <nav> element represents a section of a page that links to other pages or to parts within the page: a section with navigation links.
S - SECTION
The <section> element is a structural HTML element used to group together related elements. Each <section> typically includes one or more heading elements and additional elements presenting related content.
T - TABLE
To create table we use table tag.
A - ARTICLE
The <article> tag specifies independent, self-contained content.
F - FOOTER
The <footer> element is a structural element used to identify the footer of a page, document, article, or section. A <footer> typically contains copyright and authorship information or navigational elements pertaining to the contents of the parent element.
F - FORM
To create form in HTML we use form tag.

Image Handling

The HTML <img> tag is used to embed an image in a web page.

Images are not technically inserted into a web page; images are linked to web pages. The <img> tag creates a holding space for the referenced image.

The <img> tag is empty, it contains attributes only, and does not have a closing tag.

The <img> tag has two required attributes:

  • src - Specifies the path to the image
  • alt - The required alt attribute provides an alternate text for an image, if the user for some reason cannot view it (because of slow connection, an error in the src attribute)
  • height and width attribute - to set height and width of image according to our requirement we will use these attributes

Absolute URL vs Relative URL

A full web address in the src attribute:

<img src="https://www.w3.organisation.com">

A local link (a link to a page within the same website) is specified with a relative URL (without the "https://www" part):

<img src="imgSvg" id="luzwx22223333333334" src="data:image/svg+xml;base64,...">

Note: The data URI example is truncated; typically, you’d replace it with a relative path like "/images/example.jpg".

Anchor Tags

The "href" attribute is the most important attribute of the anchor <a> tag. and which links to destination page or URL.

Syntax of HTML anchor Tag given below:

<a href=".....">
    link text
</a>

Appearance of anchor links:

By default, links will appear as follows in all browsers:

  • An unvisited link is underlined and purple
  • A visited link is underlined and blue
  • An active link is underlined and red

Ways to link anchor tag:

  • Text as a Link: To use a text as a link, just write the text inside the <a> tag with href attribute
  • Image as a Link: To use an image as a link, just put the <img> tag inside the <a> tag
  • Button as a Link: To use a button as a link, just put the <button> tag inside the <a> tag

Target Attribute in Anchor Tag

By default, the linked page will be displayed in the current browser tab. To change this, you must specify another attribute that is target for the link.

The target attribute specifies where to open the linked document.

The target attribute can have one of the following values:

  • _self - Default. Opens the document in the same window/tab as it was clicked
  • _blank - Opens the document in a new window or tab
  • _top - Opens the document in top of window
  • _parent - Opens the document in parent web page.

Form Creation

The <form> Element

The HTML <form> element is used to create an HTML form for user input

The <form> element is a container for different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.

The <input> Element

The HTML <input> element is the most used form element.

An <input> element can be displayed in many ways, depending on the type attribute.

Example:

<input type="text">
<input type="radio">
<input type="checkbox">
<input type="tel">
<input type="password"> and so on..........

Basic Tags of Form Creation

  1. input tag ---- it is unpaired tag and defines an input control and also an type
  2. label tag ---- to mention a label for respective input
  3. textarea ---- Defines a multiline input control (text area)
  4. fieldset ---- Groups related elements in a form
  5. legend ---- Defines a caption for a <fieldset> element
  6. select ---- Defines a choosable drop-down list
  7. optgroup ---- Defines a group of related options in a drop-down list
  8. option ---- Defines an option in a drop-down list
  9. button ---- Defines a clickable button

Basic Attributes of Form

  • type ---- it will specify which type of data we should enter in the input box ex. text, number, date, tel, radio, checkbox, file, password etc.
  • for and id ---- for in label tag and id in input tag. To connect the label tag and respective input tag with it we use for and id attribute. The value should be same in both the attributes
  • name ---- the attribute specifies the result in navigator after submission of form
  • value ---- the attribute specifies the validation for type (radio) where you have to choose any one option or multiple options (checkbox)

Validation Attributes of Form

  • readonly: The input readonly attribute specifies that an input is read-only field which means we cannot write inside that input but we can click
  • disabled: The input disabled attribute specifies that an input field is disabled. A disabled input field is unusable and un-clickable.
  • size: The input size attribute specifies the visible width, in characters, of an input field. The default size is 20. Note: The size attribute works with the following input types: text, tel, email, and password.
  • maxlength: The input maxlength attribute specifies the maximum number of characters allowed in an input field.
  • min & max: The input min and max attributes specify the minimum and maximum values for an input field.
  • placeholder: The input placeholder attribute specifies a short hint that describes the expected value of an input field. It shows the faint or false text in input box and whenever you will enter the data it will disappear
  • required: The input required attribute specifies that an input field must be filled out before submitting the form.
  • autofocus: By using this attribute whenever you will load the webpage the targeted input field get automatically focused
  • checked: The checked attribute is a When present, it specifies that an <input> element should be pre-selected (checked) when the page loads.
  • pattern: This attribute specifies the valuable password for password type and it has syntax like pattern="[A-Za-z0-9]{}"

3 Ways to Add CSS in HTML

  1. Inline CSS
  2. Internal CSS
  3. External CSS

Inline CSS

The code is written in the opening tag of the HTML element by using style attribute.

Syntax:

<p style="property : value"> content </p>

Internal CSS

The code is written inside or after head tag section of an HTML element by using <style> tag.

Syntax:

<head>
    <style>
        tag_name or selector {
            property: value;
        }
    </style>
</head>

External CSS

External CSS styling can be done by creating an external CSS file with file_name.css extension and providing the link between that CSS file to HTML file by <link> tag in the head section of HTML element.

Syntax:

<head>
    <link rel="stylesheet" href="file_name.css">
</head>

NOTE:

rel: specifies the relation. rel="stylesheet" No value other than "stylesheet" should be declared

CSS Selectors

Selectors are used to select or target the particular HTML element to style them. Or it is term or word which we will mention to style multiple elements at the same time

There are five types of Selectors:

  1. Simple Selectors.
  2. Combinator Selectors.
  3. Attribute Selectors.
  4. Pseudo Class Selectors.
  5. Pseudo Element Selectors.

Simple Selector

Simple selectors style the HTML element in five ways:

  1. Id Selector (#)
  2. Class Selector (. (dot))
  3. Tag name selector (tagname itself)
  4. Grouping selector (, (comma))
  5. Universal selector (*)

ID Selector

Id selector targets only individual element in the HTML document.

Prefix symbol used for id selector is # (hash) id should be unique

HTML file:

<p id="demo">HELLO</p>

CSS file:

#demo {
    color: red;
    background: yellow;
}

Class Selector

Class selectors targets multiple elements in the HTML document.

Prefix symbol used for id selector is . (dot) class can be multiples

Syntax:

HTML:

<h1 class="test">HELLO</h1>
<h2 class="test">GOOD</h2>
<h3 class="test">MORNING</h3>

CSS:

.test {
    color: magenta;
    background: green;
}

Tagname Selector

Tagname selector targets the HTML elements by tag name.

There is no symbol used in tagname selectors. We have to use tag as name itself

Syntax:

<h1>HEADING 1</h1>
<h2>HEADING 2</h2>
<h3>HEADING 3</h3>
        
h1 {
    color: magenta;
    background: green;
}
h2 {
    color: red;
    background: black;
}
h3 {
    color: yellow;
    background: orange;
}

Grouping Selector

Grouping selector targets a group of HTML elements by tagname separated by commas (,)

Syntax:

<p>Paragraph</p>
<div>Division</div>
<h4>HEADING 4</h4>
        
p, div, h4 {
    color: magenta;
    background: green;
}

Universal Selectors

Universal Selectors targets every HTML element. The symbol of Universal Selector is *

Syntax:

<p>Paragraph</p>
<div>Division</div>
<h4>HEADING 4</h4>
        
* {
    color: yellow;
    background: magenta;
}

Combinator Selector

Combinator Selectors are used to style the combination of HTML elements.

  1. Descendent Selector: This Selector targets both direct and indirect children of a parent tag. The symbol is (space)
    2. Syntax: parent_tag child_tag { property: value; }
  2. Child Selector: This Selector targets only the direct children of the parent. The symbol is > (greater than)
    3. Syntax: parent_tag > child_tag { property: value; }
  3. Adjacent Selector: This Selector targets the element which is the first sibling of the targeted HTML tag or element. (or) This Selector targets the element which is exactly adjacent to the targeted HTML tag or element. The symbol is + (plus)
    4. Syntax: first_sib_tag + second_sib_tag { property: value; }
  4. General Sibling Selector: This Selector targets all the siblings of the target HTML tag or element. (or) This Selector targets all the adjacent tags of the target HTML tag or element. The symbol is ~ (tilde)
    5. Syntax: first_sib_tag ~ second_sib_tag { property: value; }

Attribute Selectors

Attribute selectors are used to style the HTML element by targeting the respective attributes and with their values.

Types of Attribute Selector:

  1. tag_name[attribute_name]: Represents elements with an attribute name of attribute_name.
  2. tag_name[attribute_name="value"]: Represents element with an attribute name of attribute_name whose value is exactly value.

Pseudo Elements Selectors

For pseudo elements selectors will target by (::) this symbol

  1. First Letter: It will target the first letter of respective content of targeted HTML element
  2. First Line: It will target the first line of respective content of targeted HTML element
  3. Before: It will add the any type of content or CSS property just before the targeted HTML element
  4. After: It will add the any type of content or CSS property right after the targeted HTML element
  5. Selection: Whenever we will select any kind of content will get color as blue as default but by using selection selector we can change that color and background color to
  6. Marker: This selector will change the color of list marker and only it will applicable for li tag

Box Modelling

Box Model is essentially a box that wraps around every HTML element

Box Model is used to design and layout.

The CSS Box Model consists of Margin, Border, Padding and the actual Content.

  • Content: It is the content of the HTML element.
  • Padding: It is the space after the content.
  • Border: It is the Border of the HTML element.
  • Margin: It is the space after the Border of an HTML element.

Padding and Margin allow up to four values.

Syntax:

|  | T | R | B | L |
|------|-----|-----|-----|
| padding: 10px | 0 | 0 | 0 |
| padding: 10px 20px | 0 | 0 |
| padding: 10px 20px 30px | 0 |
| padding: 10px 20px 30px 40px |

Animation in CSS

An animation lets an element gradually change from one style to another. (one color to another color) You can change as many CSS properties you want, as many times as you want.

To use CSS animation, you must first specify some keyframes for the animation. Keyframes hold what styles the element will have at certain times.

Animation properties:

  • animation-name
  • animation-duration
  • animation-timing-function
  • animation-delay
  • animation-iteration-count
  • animation-direction - alternate, normal, reverse
@keyframes identifier(animation name) {
    from {
        property: value;
    }
    to {
        property: value;
    }
}

In the example above we have specified when the style will change by using the keywords "from" and "to" (which represents 0% (start) and 100% (complete)).

It is also possible to use percent. By using percent, you can add as many style changes as you like.
