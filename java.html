1. Java Basics

Java is a high-level, object-oriented programming language that runs on the JVM (Java Virtual Machine).

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Detailed Explanation:

  • JVM: Executes bytecode, making Java platform-independent.
  • Syntax: C-like with strict typing.
  • Main Method: Entry point of a program.
  • Compilation: .java â†’ .class via javac.
Java is "write once, run anywhere" due to JVM.

2. Variables

Variables store data with specific types and scopes.

int age = 25;           // Primitive
double salary = 5000.5; // Primitive
String name = "John";   // Reference type
final int MAX = 100;    // Constant

public class Example {
    static int globalVar = 10; // Class scope
    void method() {
        int localVar = 5;     // Method scope
    }
}

Types:

  • Primitive: int, double, char, boolean
  • Reference: String, arrays, objects
  • Scope: Local, instance, static
  • final: Immutable values

3. Control Flow

Controls program execution with conditions and loops.

int x = 10;
if (x > 0) {
    System.out.println("Positive");
} else if (x == 0) {
    System.out.println("Zero");
} else {
    System.out.println("Negative");
}

for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

Structures:

  • if-else: Conditional branching
  • for: Fixed iteration
  • while/do-while: Condition-based loops
  • switch: Multi-case selection

4. Object-Oriented Programming (OOP)

Java is built around classes and objects.

public class Person {
    String name;
    int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    void display() {
        System.out.println(name + ", " + age);
    }
}

Person p = new Person("Alice", 30);
p.display();

Concepts:

  • Class: Blueprint
  • Object: Instance of class
  • Constructor: Initializes objects
  • this: Refers to current instance
OOP promotes reusability and modularity.

5. Inheritance

Allows classes to inherit properties and methods.

class Animal {
    void eat() {
        System.out.println("Eating...");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Barking...");
    }
}

Dog d = new Dog();
d.eat();  // Inherited
d.bark(); // Own method

Features:

  • extends: Inherits from parent
  • super: Access parent class
  • Single Inheritance: One parent only

6. Polymorphism

Methods can behave differently based on object type.

class Animal {
    void sound() {
        System.out.println("Some sound");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("Meow");
    }
}

Animal a = new Cat(); // Upcasting
a.sound(); // Outputs "Meow"

Types:

  • Overriding: Redefine parent method
  • Overloading: Same name, different parameters
  • Upcasting: Treat subclass as parent

7. Encapsulation

Protects data using access modifiers and getters/setters.

public class Student {
    private String name; // Encapsulated
    private int age;
    
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
}

Student s = new Student();
s.setName("Bob");

Modifiers:

  • private: Class-only access
  • protected: Package + subclasses
  • public: Everywhere
  • default: Package-only if unspecified

8. Exceptions

Handles runtime errors gracefully.

try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Error: " + e.getMessage());
} finally {
    System.out.println("Done");
}

Features:

  • try-catch: Handle exceptions
  • finally: Always executes
  • throws: Declare exceptions
  • throw: Manually throw exceptions

9. Collections

Framework for storing and manipulating groups of objects.

import java.util.*;

List list = new ArrayList<>();
list.add("Apple");

Set set = new HashSet<>();
set.add(1);

Map map = new HashMap<>();
map.put("One", 1);

Key Interfaces:

  • List: Ordered, allows duplicates
  • Set: No duplicates
  • Map: Key-value pairs
Use generics for type safety.

10. Threads

Enables concurrent execution.

class MyThread extends Thread {
    public void run() {
        System.out.println("Thread running");
    }
}

MyThread t = new MyThread();
t.start();

Runnable r = () -> System.out.println("Runnable");
Thread t2 = new Thread(r);
t2.start();

Approaches:

  • Extend Thread: Inherit and override run()
  • Implement Runnable: More flexible
  • synchronized: Thread safety
