1. SQL Basics

SQL (Structured Query Language) is a standard language for managing and manipulating relational databases.

SELECT * FROM employees;

Detailed Explanation:

  • SELECT: Retrieves data from a table.
  • FROM: Specifies the table to query.
  • WHERE: Filters rows based on conditions.
  • ORDER BY: Sorts the result set.
SQL is case-insensitive, but keywords are conventionally written in uppercase.

2. DDL (Data Definition Language)

DDL commands are used to define and modify database structures like tables and schemas.

CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    salary DECIMAL(10, 2)
);

ALTER TABLE employees ADD COLUMN department VARCHAR(50);

DROP TABLE employees;

Commands:

  • CREATE: Creates new tables, databases, or indexes.
  • ALTER: Modifies existing structures (e.g., adding columns).
  • DROP: Deletes structures from the database.

3. DML (Data Manipulation Language)

DML commands manipulate the data stored within tables.

INSERT INTO employees (id, name, salary) VALUES (1, 'Alice', 50000);

UPDATE employees SET salary = 55000 WHERE id = 1;

DELETE FROM employees WHERE id = 1;

Commands:

  • INSERT: Adds new records to a table.
  • UPDATE: Modifies existing records.
  • DELETE: Removes records from a table.

4. Constraints

Constraints enforce rules on the data in tables to maintain integrity.

CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
);

ALTER TABLE employees ADD CONSTRAINT unique_name UNIQUE (name);

Types:

  • PRIMARY KEY: Uniquely identifies each record.
  • FOREIGN KEY: Links two tables together.
  • UNIQUE: Ensures all values in a column are unique.
  • NOT NULL: Prevents null values in a column.

5. Joins

Joins combine data from multiple tables based on related columns.

SELECT e.name, d.department_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;

SELECT e.name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.department_id = d.id;

Types:

  • INNER JOIN: Returns rows with matching values in both tables.
  • LEFT/RIGHT JOIN: Includes all rows from one table, with matches from the other.
  • FULL JOIN: Includes all rows from both tables, with NULLs where no match exists.
  • SELF JOIN: Joins a table to itself.

6. Subqueries

Subqueries are queries nested within another query to perform complex operations.

SELECT name FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

SELECT * FROM orders
WHERE customer_id IN (SELECT id FROM customers WHERE country = 'USA');

Features:

  • Scalar Subquery: Returns a single value.
  • Row Subquery: Returns a single row.
  • Table Subquery: Returns multiple rows.

7. Indexes

Indexes improve the speed of data retrieval operations on a table.

CREATE INDEX idx_name ON employees (name);

DROP INDEX idx_name;

Features:

  • CREATE INDEX: Creates an index on a column.
  • DROP INDEX: Removes an index.
  • Clustered/Non-clustered: Types of indexes affecting data storage.
Indexes speed up SELECT queries but may slow down INSERT, UPDATE, and DELETE operations.

8. Transactions

Transactions ensure data integrity by grouping operations with ACID properties.

BEGIN TRANSACTION;

UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;

COMMIT;

Properties:

  • Atomicity: Ensures all operations complete or none do.
  • Consistency: Maintains a valid database state.
  • Isolation: Ensures transactions are independent.
  • Durability: Guarantees changes persist after commit.

9. Views

Views are virtual tables created from the result of a query.

CREATE VIEW high_salary_employees AS
SELECT * FROM employees WHERE salary > 60000;

SELECT * FROM high_salary_employees;

Features:

  • CREATE VIEW: Defines a new view.
  • DROP VIEW: Removes a view.
  • Materialized Views: Physically store query results (database-specific).

10. Stored Procedures

Stored procedures are reusable SQL scripts stored in the database.

CREATE PROCEDURE increase_salary(IN emp_id INT, IN amount DECIMAL)
BEGIN
    UPDATE employees SET salary = salary + amount WHERE id = emp_id;
END;

CALL increase_salary(1, 5000);

Features:

  • CREATE PROCEDURE: Defines a new procedure.
  • CALL: Executes a stored procedure.
  • Parameters: Supports IN, OUT, and INOUT types.
